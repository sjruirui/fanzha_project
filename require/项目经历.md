# 反诈宣传平台 - 项目经历

## 项目概述

反诈宣传平台是一个综合性的反电信网络诈骗宣传教育平台，集 AI 反诈骗助手、AI 情景模拟、反诈资讯发布、知识科普、互动交流、在线学习、风险测试、案件举报等功能于一体。平台采用前后端分离架构，分为用户端和管理端，为用户提供全方位的反诈知识学习和互动体验，为管理员提供高效的内容管理和数据分析能力。

**主要功能模块：**

| 模块 | 功能描述 |
|------|----------|
| AI反诈助手 | 智能问答、情景模拟（冒充公检法、刷单返利、杀猪盘、虚假网贷） |
| 反诈资讯 | 资讯浏览、分类筛选、关键词搜索、点赞收藏评论 |
| 反诈知识库 | 知识分类浏览、人群分类（老年人、学生、企业员工等） |
| 反诈课堂 | 章节化视频学习、课时管理、学习进度跟踪 |
| 交流互动 | 帖子发布、分类管理、评论回复、审核机制 |
| 活动中心 | 线上/线下活动发布、报名管理、互动交流 |
| 反诈自测 | 闯关答题模式、单选/多选题型、成绩记录 |
| 诈骗举报 | 在线举报提交、证据上传、状态跟踪 |
| 个人中心 | 个人资料管理、发帖记录、活动记录、点赞收藏历史 |
| 管理后台 | 用户管理、内容管理、数据统计、举报处理 |

## 项目亮点

### 1. AI智能反诈助手（核心亮点）

**技术实现：**
- 集成通义千问大模型，实现智能问答和情景模拟两种模式
- 采用 SSE（Server-Sent Events）技术实现流式响应，提升用户体验
- 情景模拟模式支持四种典型诈骗场景：冒充公检法、刷单返利、杀猪盘、虚假网贷
- AI扮演诈骗者与用户对话，帮助用户识别诈骗手法

**技术难点：**
- SSE流式传输的实现与前端实时渲染
- 会话状态管理与上下文保持
- 多情景模式的动态切换

### 2. 前后端分离架构

**架构设计：**
- 用户端与管理端共享同一Vue应用，通过路由前缀（`/` 和 `/admin`）区分
- 采用路由守卫实现权限控制，未登录用户自动弹出登录弹窗
- API统一响应格式 `{ code, message, data }`，便于前端统一处理
- JWT Token认证机制，前端拦截器自动携带Token

**优势：**
- 减少代码重复，提高开发效率
- 统一的错误处理和响应格式
- 安全的认证机制

### 3. 丰富的互动功能

**功能实现：**
- 点赞/收藏/评论系统，支持帖子、活动、资讯、知识四种目标类型
- 评论支持多级回复，实现嵌套评论结构
- 用户活动报名与取消，状态实时更新
- 帖子审核机制，管理员可审核通过或拒绝

**技术细节：**
- targetType枚举设计，统一处理不同类型的互动
- 评论parentId和replyToUserId实现回复链
- 报名状态检查接口，防止重复报名

### 4. 闯关式反诈自测

**设计思路：**
- 采用关卡模式，增加学习趣味性
- 支持单选题和多选题两种题型
- 答题提交后即时反馈正确答案和解析
- 记录答题历史，便于用户回顾学习

**技术实现：**
- 关卡与题目分离存储，便于灵活配置
- 答题结果计算：正确数、总分、是否通过
- 错题解析展示，强化学习效果

### 5. 完善的管理后台

**功能模块：**
- 数据统计：用户数、帖子数、活动数、举报数、趋势图
- 内容管理：资讯、知识、课堂、帖子、活动、公告
- 用户管理：用户列表、管理员管理、批量操作
- 举报处理：状态流转、备注处理、证据查看
- 评论管理：评论审核、批量删除

**技术亮点：**
- 分页查询统一封装，减少重复代码
- 批量删除接口，提高操作效率
- 状态枚举设计，清晰的状态流转

### 6. 数据库设计

**核心表结构：**
- 用户表：普通用户、管理员分离存储
- 内容表：资讯、知识、课堂章节课时、帖子
- 互动表：点赞、收藏、评论、报名
- 测试表：关卡、题目、答题记录
- 举报表：举报信息、证据、处理状态

**设计原则：**
- 合理的字段命名和数据类型
- 外键关联保证数据一致性
- 状态字段统一管理（0禁用/待审核，1启用/已通过）
