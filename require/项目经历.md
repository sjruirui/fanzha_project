# 反诈宣传平台 - 项目经历

## 项目概述

反诈宣传平台是一个综合性的反电信网络诈骗宣传教育平台，集 AI 反诈骗助手、AI 情景模拟、反诈资讯发布、知识科普、互动交流、在线学习、风险测试、案件举报等功能于一体。平台采用前后端分离架构，分为用户端和管理端，为用户提供全方位的反诈知识学习和互动体验，为管理员提供高效的内容管理和数据分析能力。

**主要功能模块：**

| 模块 | 功能描述 |
|------|----------|
| AI反诈助手 | 智能问答、情景模拟（冒充公检法、刷单返利、杀猪盘、虚假网贷） |
| 反诈资讯 | 资讯浏览、分类筛选、关键词搜索、点赞收藏评论 |
| 反诈知识库 | 知识分类浏览、人群分类（老年人、学生、企业员工等） |
| 反诈课堂 | 章节化视频学习、课时管理、学习进度跟踪 |
| 交流互动 | 帖子发布、分类管理、评论回复、审核机制 |
| 活动中心 | 线上/线下活动发布、报名管理、互动交流 |
| 反诈自测 | 闯关答题模式、单选/多选题型、成绩记录 |
| 诈骗举报 | 在线举报提交、证据上传、状态跟踪 |
| 个人中心 | 个人资料管理、发帖记录、活动记录、点赞收藏历史 |
| 管理后台 | 用户管理、内容管理、数据统计、举报处理 |

## 项目亮点

### 1. 构建优化 - Gzip 压缩与代码分割

**技术实现：**
- 使用 `vite-plugin-compression` 实现 Gzip 压缩，大于 10KB 的文件才压缩
- 压缩效果显著：element-plus.js 从 975KB 压缩至 314KB，vue-vendor.js 从 274KB 压缩至 70KB
- 采用 Vite 8 的 `manualChunks` 函数配置代码分割策略
- 分离策略：element-plus、vue-vendor（Vue+Pinia+Vue Router）、editor（WangEditor）

**技术难点：**
- Vite 8 使用 Rolldown 替代 Rollup，需使用函数形式的 manualChunks
- 兼容 Rolldown 的代码分割配置

### 2. 图片懒加载

**技术实现：**
- 创建自定义 Vue 指令 `v-lazy`，使用 Intersection Observer API 检测元素进入视口
- 支持加载状态样式切换（lazy-loading、lazy-loaded、lazy-error）
- 应用于首页、资讯、知识、活动、社区、课堂等所有列表页面

**技术优势：**
- 原生 Intersection Observer API，性能优于滚动监听
- 减少首屏图片请求数量，提升页面加载速度
- 平滑的淡入动画效果

### 3. 骨架屏加载状态

**技术实现：**
- 创建通用骨架屏组件：Skeleton（基础）、SkeletonCard（卡片网格）、SkeletonList（列表带头像）
- 使用 CSS 动画模拟加载效果，提升用户体验
- 应用于所有列表页面的数据加载状态

**用户体验提升：**
- 避免页面空白，提供即时视觉反馈
- 骨架屏布局与实际内容一致，减少布局跳动

### 4. AI智能反诈助手（核心亮点）

**技术实现：**
- 集成通义千问大模型，实现智能问答和情景模拟两种模式
- 采用 SSE（Server-Sent Events）技术实现流式响应，提升用户体验
- 情景模拟模式支持四种典型诈骗场景：冒充公检法、刷单返利、杀猪盘、虚假网贷
- AI扮演诈骗者与用户对话，帮助用户识别诈骗手法

**技术难点：**
- SSE流式传输的实现与前端实时渲染
- 会话状态管理与上下文保持
- 多情景模式的动态切换

### 5. 前后端分离架构

**架构设计：**
- 用户端与管理端共享同一Vue应用，通过路由前缀（`/` 和 `/admin`）区分
- 采用路由守卫实现权限控制，未登录用户自动弹出登录弹窗
- API统一响应格式 `{ code, message, data }`，便于前端统一处理
- JWT Token认证机制，前端拦截器自动携带Token

**优势：**
- 减少代码重复，提高开发效率
- 统一的错误处理和响应格式
- 安全的认证机制

### 6. Element Plus 3.0 兼容性处理

**问题修复：**
- el-radio-button 的 `label` 属性弃用，改用 `value` 属性
- el-pagination 的 `small` 属性弃用，改用 `size="small"`
- 统一处理空字符串值问题，避免 API 弃用警告

**技术细节：**
- 将空字符串 `''` 或 `undefined` 值改为数字 `0`，API 调用时再转换
- 保证代码向前兼容 Element Plus 3.0 版本
